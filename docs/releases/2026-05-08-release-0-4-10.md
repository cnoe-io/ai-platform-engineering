> Released: 2026-05-08
> Chart: `oci://ghcr.io/cnoe-io/charts/ai-platform-engineering:0.4.10`
> Previous release: 0.4.9

## Highlights

0.4.10 introduces server-side boilerplate injection for the Overthink reasoning engine — operators can now define a global prompt template without touching individual agent configs. VictorOps moves fully to the v2 incidents API, and the Supervisor Skills admin panel gets improved badge labels and tooltips. A batch of quick-install UX fixes and Helm ServiceMonitor hardening round out the release.

<!-- truncate -->

## What's New

### Overthink Server-Side Boilerplate
- **`OVERTHINK_BOILERPLATE` template variable** — define a global reasoning preamble once in the server environment; all Overthink-enabled agents inherit it automatically ([#1364](https://github.com/cnoe-io/ai-platform-engineering/pull/1364))
- **Slack-bot injection via `client_context`** — the boilerplate is now injected at the Slack bot layer rather than duplicated per-agent, keeping agent configs clean ([#1369](https://github.com/cnoe-io/ai-platform-engineering/pull/1369))

### VictorOps v2 Incidents API
- **Consolidated to v2 reporting** — all incident creation and update calls now use the VictorOps v2 incidents endpoint, dropping the legacy slim projections ([#1393](https://github.com/cnoe-io/ai-platform-engineering/pull/1393))

### Admin: Supervisor Skills UX
- **Improved badge labels and tooltips** — the Skills panel in the Supervisor admin UI shows clearer status badges and hover tooltips for each installed skill ([#1394](https://github.com/cnoe-io/ai-platform-engineering/pull/1394))

### Skills & Gateway Quick Install
- `HIDE_BUILTIN_SKILLS` flag — operators can hide the default built-in skills from the catalog, showing only custom agent skills ([#1394](https://github.com/cnoe-io/ai-platform-engineering/pull/1394))
- Catalog key bypass for `/skills/refresh` — lets the skills refresh endpoint work without a full auth token in locked-down environments
- Multiple UX improvements to the Gateway quick-install flow: simplified options, clearer API key gate, corrected launch guidance, and removed pagination from catalog URL ([#1382](https://github.com/cnoe-io/ai-platform-engineering/pull/1382), [#1383](https://github.com/cnoe-io/ai-platform-engineering/pull/1383))

## Bug Fixes

## Upgrade Guide: 0.4.9 → 0.4.10

### Overview

Drop-in upgrade — no `values.yaml` edits required. New Overthink boilerplate behavior is opt-in via environment variable.

### Helm Values Changes

No breaking Helm values changes.

#### New Optional: Overthink Boilerplate

To set a global reasoning preamble for all Overthink agents, add to your values:

```yaml
supervisor:
  config:
    OVERTHINK_BOILERPLATE: |
      You are a platform engineering assistant. Always prefer Kubernetes-native solutions.
```

Leave unset to preserve existing behavior.

### Upgrade Runbook

```bash
helm upgrade ai-platform-engineering \
  oci://ghcr.io/cnoe-io/charts/ai-platform-engineering \
  --version 0.4.10 \
  -f your-values.yaml

kubectl get pods -n <namespace>
```
