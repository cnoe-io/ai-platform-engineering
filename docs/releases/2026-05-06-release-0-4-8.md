CAIPE 0.4.8 is out. This release adds a brand-new AWS MCP server, configurable call-limit middlewares to prevent runaway agents, and Kubernetes Pod Security Standards Baseline compliance across all Helm charts. Several long-standing bugs are also fixed — including a silent OTLP trace drop issue and an MCP trailing-slash redirect problem.

<!-- truncate -->

## Highlights

### 🆕 AWS MCP Server

A new `mcp-aws` agent exposes `aws_cli_execute` and `eks_kubectl_execute` as MCP tools over HTTP/SSE. Built on FastMCP, it supports opt-in `INSTALL_AWS_CLI=true` and `INSTALL_KUBECTL=true` Docker build args and is automatically built as `ghcr.io/cnoe-io/mcp-aws`.

### 🛡️ Supervisor Call-Limit Middlewares

Two new opt-in middlewares cap per-run resource usage:

| Env Var | Default | Purpose |
|---------|---------|---------|
| `TOOL_CALL_LIMIT` | `0` (off) | Cap total tool invocations per run |
| `MODEL_CALL_LIMIT` | `0` (off) | Cap total LLM inference calls per run |

An optional `SummarizationMiddleware` compresses conversation history when approaching token limits.

### 🔐 Kubernetes PSS Baseline

All Helm chart subcharts now set `securityContext` values satisfying the [Kubernetes PSS Baseline profile](https://kubernetes.io/docs/concepts/security/pod-security-standards/). No `values.yaml` changes needed.

### 📡 Skills Gateway Improvements

- Admin scan override — trigger a skills scan without waiting for the scheduled interval
- GitLab tree API pagination (previously returned only the first page)
- GitHub truncation detection
- Live crawl console in the admin UI

## Bug Fixes

- **tracing**: New `SkillScrubberSpanProcessor` scrubs skill/workflow content from OTLP spans before export; hard cap on attribute size prevents silent 413 drops to Langfuse
- **mcp**: Remove trailing slash from default HTTP MCP path — prevents 307 redirect failures on startup
- **ui**: Suppress synthetic `Task <status> (ID: ...)` filler text in chat
- **slack-bot**: Fix subteam/usergroup mention syntax (`<!subteam^{id}>` vs `<@{id}>`)
- **setup**: Replace bash 4+ case conversion with POSIX-compatible `tr` for macOS bash 3.2
