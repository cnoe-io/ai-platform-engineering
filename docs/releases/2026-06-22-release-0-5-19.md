> Released: 2026-06-22
> Chart: `oci://ghcr.io/cnoe-io/charts/ai-platform-engineering:0.5.19`
> Previous release: 0.5.18

## Highlights

0.5.19 is a targeted bug-fix release. It removes an unnecessary Okta API call that fired on every Identity Sync page load and was causing rate limit violations on shared API tokens. It also fixes the Helm chart to correctly scope the audit service URL to the release namespace so multi-tenant installs stop routing audit events to the wrong service.

<!-- truncate -->

## Bug Fixes

- **identity-sync**: Remove proactive Okta credential health check on page load — the `GET /api/v1/groups?limit=1` probe that ran each time the Identity Sync tab opened triggered `system.operation.rate_limit.violation` events in the Okta audit log for shared tokens. Credential status now shows **Configured** (env vars present, local check) or **Not set** (unconfigured) with no outbound Okta call. Sync errors still surface on the run row in Sync History. ([#1975](https://github.com/cnoe-io/ai-platform-engineering/pull/1975))
- **charts**: Default `AUDIT_SERVICE_URL` to a release-scoped audit service endpoint — previously the value pointed at a generic hostname that could resolve to another release in multi-tenant clusters, misdirecting audit events.

## Breaking Changes

No breaking changes. Drop-in upgrade from 0.5.18.

```bash
  oci://ghcr.io/cnoe-io/charts/ai-platform-engineering \

```

No Helm values changes between 0.5.18 and 0.5.19. No data migrations required.
