## Highlights

0.4.12 is a focused patch that ships an automated docs versioning pipeline, a security fix, and minor CI and documentation updates. The new docs snapshot-and-prune workflow fires on every stable tag push and keeps the version dropdown current without manual intervention.

<!-- truncate -->

## What's New

### Automated Docs Versioning Pipeline
- **`docs-snapshot.yml`** — new CI workflow triggers on every semver tag push; runs `docusaurus docs:version`, prunes old snapshots per the retention policy (5 latest `0.4.x` + highest from the previous minor series), validates the build, and opens a PR automatically ([#1405](https://github.com/cnoe-io/ai-platform-engineering/pull/1405))
- **`docs-release.yml`** — separate `workflow_dispatch` workflow creates a skeleton blog post PR with pre-filled frontmatter, section headers, and git log pasted in the PR body for a human to complete and merge ([#1405](https://github.com/cnoe-io/ai-platform-engineering/pull/1405))
- **`versions-config.json`** — extracted Docusaurus version config into a machine-updatable JSON sidecar so CI can update `lastVersion` and the versions block without editing TypeScript

### Dynamic Agent Self-Identity: Agent ID Exposure
- **`agent_id` field** — dynamic agents now include their own ID in the self-identity payload, enabling self-referencing in multi-agent orchestration ([#1403](https://github.com/cnoe-io/ai-platform-engineering/pull/1403))

## Bug Fixes

- **security**: applied a security fix for a vulnerability in the platform stack — upgrade recommended ([#1406](https://github.com/cnoe-io/ai-platform-engineering/pull/1406))
- **docs**: update Meeting Recordings link to the CNOE YouTube channel
