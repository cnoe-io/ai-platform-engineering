> Previous release: 0.4.10

## Highlights

0.4.11 ships the LiteLLM MCP server and FinOps reporting integration — operators can now deploy a cost-tracking MCP alongside the existing LiteLLM proxy and get model spend reports as workflow outputs. Dynamic agents gain a self-identity API that exposes the agent's own ID at runtime. The Overthink reasoning engine gets a rewritten stepwise boilerplate prompt, and CI gains a manual prebuild trigger for PRs with merge conflicts.

## What's New

### LiteLLM MCP Server & FinOps Integration
- **`litellm-mcp` server** — new MCP server that wraps the LiteLLM proxy, enabling agents to call LiteLLM tools via the MCP protocol ([#1402](https://github.com/cnoe-io/ai-platform-engineering/pull/1402))
- **LiteLLM FinOps MCP deployment** — Helm subchart for the FinOps reporting MCP; integrates with the existing LiteLLM spend tracking ([#1402](https://github.com/cnoe-io/ai-platform-engineering/pull/1402))
- **Improved FinOps report exports** — richer CSV/JSON export format for model spend reports
- **Theme-aware FinOps workflow images** — report charts now respect light/dark mode

### Dynamic Agent Self-Identity
- **`agent_id` in self-identity response** — dynamic agents now expose their own agent ID via the self-identity endpoint, enabling agents to reference themselves in orchestration workflows ([#1403](https://github.com/cnoe-io/ai-platform-engineering/pull/1403))
- Updated docs to clarify self-identity exclusions (the agent ID field is not included in identity assertions to other agents)

### Overthink Boilerplate Rewrite
- **Stepwise prompt with explicit precedence** — the default Overthink boilerplate is rewritten as a numbered, stepwise reasoning protocol with explicit priority ordering, replacing the previous vague "the bot" phrasing ([#1386](https://github.com/cnoe-io/ai-platform-engineering/pull/1386))

### CI Improvements
- **Manual prebuild trigger for conflict PRs** — workflow dispatch now lets maintainers manually trigger prebuild image builds for PRs that have merge conflicts, unblocking multi-PR rebases ([#1408](https://github.com/cnoe-io/ai-platform-engineering/pull/1408))
- **Chart version bump gating** — chart versions are no longer bumped automatically on every PR; a maintainer approval step is now required ([#1408](https://github.com/cnoe-io/ai-platform-engineering/pull/1408))
- **Unified prebuild comments** — CI now posts a single consolidated comment per PR with all image build statuses instead of one comment per image ([#1376](https://github.com/cnoe-io/ai-platform-engineering/pull/1376))

## Bug Fixes

- **auth**: restrict catalog key bypass to `/skills/refresh` path only — previously the bypass was too broad ([#1389](https://github.com/cnoe-io/ai-platform-engineering/pull/1389))
- **ci**: use local git tags for version lookup instead of relying on remote fetch, fixing version resolution in isolated runners ([#1406](https://github.com/cnoe-io/ai-platform-engineering/pull/1406))
- **litellm**: avoid secret-like example values in LiteLLM config templates that could trigger secret scanners ([#1402](https://github.com/cnoe-io/ai-platform-engineering/pull/1402))
- **community**: update Meeting Recordings link to the CNOE YouTube channel ([#1405](https://github.com/cnoe-io/ai-platform-engineering/pull/1405))

## Upgrade Guide: 0.4.10 → 0.4.11

### Overview

Drop-in upgrade — no `values.yaml` edits required. LiteLLM MCP and FinOps are new optional subcharts disabled by default.

### Helm Values Changes

No breaking Helm values changes.

#### New Optional: LiteLLM MCP Server

To enable the LiteLLM MCP server alongside an existing LiteLLM proxy deployment:

```yaml
litellm-mcp:
  enabled: true
  config:
    LITELLM_BASE_URL: "http://litellm:4000"
    LITELLM_API_KEY: "<your-key>"
```

Leave `enabled: false` (default) to skip it.

#### New Optional: LiteLLM FinOps MCP

```yaml
litellm-finops-mcp:
  enabled: true
```

### Upgrade Runbook

```bash
helm upgrade ai-platform-engineering \
  oci://ghcr.io/cnoe-io/charts/ai-platform-engineering \

  -f your-values.yaml

kubectl get pods -n <namespace>
```
