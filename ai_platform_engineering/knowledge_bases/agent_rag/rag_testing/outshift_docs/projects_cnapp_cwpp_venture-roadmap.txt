Feature Roadmap
¶
For more information on the Panoptica feature roadmap, please visit the
Panoptica Roadmap/Demos
page in Confluence.
PLG UI
¶
The Panoptica UI team and the marketing teams are currently working on improving the current UI/UX to better suite a PLG (Product Led Growth) format. This will potentially bring more customers as well as make the product easier and lighter to use while still brining in advanced features. You may find that the new
Panoptica.app
wordpress website is striving for such a format.
Data Security
¶
Panoptica is striving to bring its product into the
Data Security
market to give a "full circle" security product. The current thought process is to use OTel (Open Telemetry) and the current API Clarity feature to extract relevant data.
API Anomalies
¶
By using some ML (Machine Learning) algorithms, Panoptica will give a more
smart
anomaly detection in its API Clarity platform. This will also help find and define some false positives for the user to be aware of.
Chain Clarity
¶
A DevOps/SRE feature that will help the users become more aware and compliant with some
Supply Chain Security
standards such as the SLSA ½/3~ The other aspect of this feature is to provide the necessary tools to not only report on compliancy issues, but also be able to act on them. This may be done with combining some open source projects like
SLSA Framework
.
