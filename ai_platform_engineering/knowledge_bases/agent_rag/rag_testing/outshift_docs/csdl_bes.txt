Build Environment Security Evidence
¶
Build Environment Security (BES)
controls are meant to secure all elements involved in the building, authentication, and signing of software across all domains (cloud, embedded, or application).
BES creates specification & architecture that defines a build environment that is Hardened, Segmented, Trusted, and Monitored.
BES specification roughly follows
SLSA Framework
SLSA: Supply chain Levels for Software Artifacts
